# Weekly Schedule

| Week | Goals | Deliverables |
|---|---|---|
| Week 1 | Finalize project idea, confirm stack, define realistic scope | Project concept, initial setup decisions |
| Week 2 | Build backlog, assign roles, plan first sprint | `backlog.md`, `sprint-1-plan.md`, `team-roles.md`, `schedule.md` |
| Week 3 | Create risk register, define Git workflow, prepare templates | `risk-register.md`, PR template, issue templates |
| Week 4 | Complete backend note model and CRUD API | Backend notes app, migrations, tested API endpoints |
| Week 5 | Connect frontend to backend and display notes list | Working list view, API integration |
| Week 6 | Implement create, edit, and delete interactions | End-to-end CRUD flow |
| Week 7 | Improve responsive design and UX feedback | Refined UI, success/error states, empty states |
| Week 8 | Add deployment configuration and staging checks | Initial Vercel and Render deployment setup |
| Week 9 | Implement search or tagging based on progress | Search feature or note organization enhancement |
| Week 10 | Strengthen validation, error handling, and QA testing | Test checklist, bug fixes |
| Week 11 | Improve accessibility and interface polish | Accessibility improvements, refined styling |
| Week 12 | Conduct team review and address technical debt | Refactoring tasks, documentation updates |
| Week 13 | Finalize deployment and prepare demo-ready build | Stable live application |
| Week 14 | Complete final QA and presentation preparation | Final bug fixes, screenshots, demo script |
| Week 15 | Submit project and present outcomes | Final submission package |
